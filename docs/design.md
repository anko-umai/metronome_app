# メトロノームアプリ 設計書

## 1. 概要

音楽のリズムの基準となるメトロノームのWebアプリケーション。  
ブラウザ上で動作し、視覚的・聴覚的にリズムを確認できる。

---

## 2. 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | HTML5 / CSS3 / JavaScript (Vanilla) |
| 音声制御 | Web Audio API |
| タイマー制御 | AudioContext のスケジューリング |
| 対応ブラウザ | Chrome, Safari, Firefox, Edge (モダンブラウザ) |
| レスポンシブ | モバイル・デスクトップ両対応 |

### 技術選定理由

- **Web Audio API**: `setInterval` / `setTimeout` はJSのイベントループに依存しタイミングが不安定になる。Web Audio API の `AudioContext.currentTime` はハードウェアクロックベースで高精度なスケジューリングが可能
- **Vanilla JS**: 外部依存なしでシンプルに構成。メトロノームの機能規模ではフレームワーク不要
- **PWA 対応可能**: 将来的にService Workerを追加しオフライン対応も容易

## 3. 機能要件

### 3.1 テンポ制御

| 項目 | 仕様 |
|------|------|
| 範囲 | 40 〜 200 BPM |
| 操作方法 | スライダー + 数値直接入力 + +/- ボタン |
| 刻み幅 | 1 BPM 単位 |
| 初期値 | 120 BPM |

### 3.2 拍子設定

| 項目 | 仕様 |
|------|------|
| 選択肢 | 2拍子 / 3拍子 / 4拍子 / 5拍子 |
| 初期値 | 4拍子 |
| 操作方法 | ボタン選択式 |

### 3.3 1拍目の強調

| 種別 | 強調方法 |
|------|---------|
| 聴覚的 | 1拍目は高い音（880Hz）、他の拍は通常の音（440Hz） |
| 視覚的 | 1拍目は強調色（赤系）で大きく表示、他の拍は通常色 |

### 3.4 音符モード（リズム確認機能）

指定したテンポに対して、以下の音符の細分化でリズムを確認できる。

| 音符 | 1拍あたりの発音数 | 説明 |
|------|-------------------|------|
| 2分音符 | 0.5回 (2拍に1回) | テンポの半分の速度 |
| 4分音符 | 1回 | 基本テンポそのまま |
| 8分音符 | 2回 | テンポの2倍の速度 |
| 16分音符 | 4回 | テンポの4倍の速度 |
| 3連符 | 3回 | 1拍を3等分 |

```

│   ├── stop()         // 再生停止
```

### 5.3 スケジューリング方式

Web Audio API の先読み（Look-ahead）スケジューリングパターンを採用する。

```
┌─────────────────────────────────────────┐
│  scheduler() ← setInterval(25ms)で呼出  │
│                                         │
│  while (nextNoteTime < currentTime      │
│         + scheduleAheadTime) {          │
│    scheduleNote(nextNoteTime, ...)      │
│    nextNoteTime += calcInterval()       │
│  }                                      │
└─────────────────────────────────────────┘
```

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| scheduleAheadTime | 100ms | 先読み時間 |
| lookahead | 25ms | scheduler の呼び出し間隔 |

この方式により、JSのタイマー精度に依存せず、正確なタイミングで音を鳴らせる。

### 5.4 音声生成

Web Audio API の `OscillatorNode` を使用してクリック音を合成する。

| 音の種類 | 周波数 | 波形 | 持続時間 | 用途 |
|---------|--------|------|---------|------|
| アクセント音 | 880 Hz | sine | 60ms | 1拍目 |
| 通常音 | 440 Hz | sine | 60ms | 2拍目以降 |
| サブディビジョン音 | 330 Hz | sine | 40ms | 細分化された拍（8分・16分・3連符） |

音の生成フロー:
```
OscillatorNode → GainNode → AudioContext.destination
                  │
                  └─ エンベロープ制御（急速な減衰で短いクリック音に）
```

## 6. 音符モードの計算ロジック

基本間隔: `quarterNoteInterval = 60 / tempo` (秒)

| 音符モード | 間隔の計算 | 例 (120 BPM) |
|-----------|-----------|--------------|
| 2分音符 | `quarterNoteInterval * 2` | 1.0秒 |
| 4分音符 | `quarterNoteInterval` | 0.5秒 |
| 8分音符 | `quarterNoteInterval / 2` | 0.25秒 |
| 16分音符 | `quarterNoteInterval / 4` | 0.125秒 |
| 3連符 | `quarterNoteInterval / 3` | 0.167秒 |

### 拍カウントとサブディビジョンの関係

4分音符モードの場合、各音が1拍に対応する。  
8分音符モードの場合、1拍内に2回鳴り、1回目がメイン拍・2回目がサブディビジョンとなる。

```
4分音符 (4拍子):  |1   |2   |3   |4   |
8分音符 (4拍子):  |1 + |2 + |3 + |4 + |
16分音符(4拍子):  |1e+a|2e+a|3e+a|4e+a|
3連符  (4拍子):  |1++|2++|3++|4++|
```

## 8. 制約事項・注意点

1. **ブラウザの自動再生ポリシー**: AudioContext はユーザー操作（クリック等）後に初期化が必要
2. **バックグラウンド制限**: ブラウザタブが非アクティブの際、`setInterval` の精度が低下する可能性がある。Web Audio API のスケジューリングで緩和
3. **モバイル対応**: iOS Safari では AudioContext の `resume()` が必要な場合がある
