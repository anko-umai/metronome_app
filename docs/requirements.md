# メトロノームアプリ 要件仕様書

## 1. 概要

### 1.1 アプリ名
メトロノームアプリ (metronome_app)

### 1.2 目的
音楽のリズムの基準となるメトロノームをブラウザ上で提供し、楽器演奏や音楽練習時の正確なテンポ維持とリズム練習を支援する。

### 1.3 対象ユーザー
- 楽器演奏者（初心者〜上級者）
- 音楽を学習する学生
- リズム練習をしたい全ての人

### 1.4 利用環境
- モダンブラウザ（Chrome / Safari / Firefox / Edge）
- デスクトップ・タブレット・スマートフォン

## 2. 機能要件

### 2.1 テンポ制御機能

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-01 |
| 機能名 | テンポ設定 |
| 範囲 | 40 〜 200 BPM |
| 初期値 | 120 BPM |
| 刻み幅 | 1 BPM |
| 操作方法 | スライダー / +ボタン / −ボタン |
| 表示 | 現在のBPM値を数値で表示 |

**詳細仕様:**
- 範囲外の値を入力した場合は上限/下限にクランプする
- 再生中もリアルタイムでテンポ変更可能

### 2.2 拍子設定機能

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-02 |
| 機能名 | 拍子選択 |
| 選択肢 | 2拍子 / 3拍子 / 4拍子 / 5拍子 |
| 初期値 | 4拍子 |
| 操作方法 | ボタン選択式（排他選択） |

**詳細仕様:**
- 選択中の拍子はビジュアル上でアクティブ状態として表示する
- 拍子変更時はビートインジケーターの数が自動的に更新される

### 2.3 1拍目強調機能

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-03 |
| 機能名 | 1拍目の聴覚・視覚強調 |

**聴覚的強調:**
| 音種 | 周波数 | 持続時間 | 用途 |
|------|-------|---------|------|
| アクセント音 | 880 Hz | 60 ms | 1拍目 |
| 通常音 | 440 Hz | 60 ms | 2拍目以降 |

**視覚的強調:**
- 1拍目: 赤色のインジケーター（サイズ大）
- 2拍目以降: 青色のインジケーター（サイズ通常）
- 現在の拍のインジケーターが点灯する

### 2.4 リズム確認機能（音符モード）

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-04 |
| 機能名 | 音符モード切替 |
| 初期値 | 4分音符 |
| 操作方法 | ボタン選択式（排他選択） |

**音符モード一覧:**

| 音符 | 1拍あたり発音数 | 間隔（120BPM時） |
|------|----------------|-----------------|
| 2分音符 | 0.5回 | 1.000 秒 |
| 4分音符 | 1回 | 0.500 秒 |
| 8分音符 | 2回 | 0.250 秒 |
| 16分音符 | 4回 | 0.125 秒 |
| 3連符 | 3回 | 0.167 秒 |

**詳細仕様:**
- 細分化された拍（8分・16分・3連符のサブディビジョン）は、メイン拍とは異なる音（330Hz、音量40%）で再生される
- 音符モードを変更すると、拍位置は1拍目にリセットされる

### 2.5 再生制御機能

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-05 |
| 機能名 | 再生・停止 |
| 操作方法 | START/STOPボタン（トグル） |

**詳細仕様:**
- 再生中はボタンの表示と色が「STOP」（赤系）に変わる
- 停止中はボタンの表示と色が「START」（緑系）になる
- 停止するとビートインジケーターの点灯はクリアされる

### 2.6 ビジュアルインジケーター機能

| 項目 | 仕様 |
|------|------|
| 要件ID | FR-06 |
| 機能名 | 拍のビジュアル表示 |

**詳細仕様:**
- 拍数に応じた数の丸いインジケーターを横並びで表示
- 音の再生タイミングに合わせてインジケーターが点灯
- メイン拍のみ点灯（サブディビジョンでは点灯変化なし）

## 3. 非機能要件

### 3.1 性能要件

| 要件ID | 項目 | 仕様 |
|--------|------|------|
| NFR-01 | リズム精度 | Web Audio API の先読みスケジューリング（100ms先読み / 25ms間隔）により、JS タイマーの精度に依存しない正確なリズムを実現する |
| NFR-02 | 応答性 | ボタン操作から100ms以内に視覚フィードバックを返す |
| NFR-03 | 起動時間 | 初回読み込み後1秒以内に操作可能 |

### 3.2 UI/UX要件

| 要件ID | 項目 | 仕様 |
|--------|------|------|
| NFR-04 | レスポンシブ対応 | デスクトップ（420px〜）・タブレット・モバイル（320px〜）で適切に表示 |
| NFR-05 | タッチ操作対応 | スマートフォンでの操作を想定し、ボタンは最低44pxのタップ領域を確保 |
| NFR-06 | カラースキーム | ダークテーマ（背景 #1a1a2e） |
| NFR-07 | アクセシビリティ | 明確なコントラスト比、アクセント色（#ff8c00）で重要要素を強調 |

### 3.3 互換性要件

| 要件ID | 項目 | 仕様 |
|--------|------|------|
| NFR-08 | ブラウザ対応 | Chrome / Safari / Firefox / Edge の最新2バージョン |
| NFR-09 | デバイス対応 | iOS 14+ / Android 8+ のスマートフォン |

### 3.4 セキュリティ要件

| 要件ID | 項目 | 仕様 |
|--------|------|------|
| NFR-10 | データ収集 | ユーザーの個人情報を一切収集しない |
| NFR-11 | 外部通信 | 外部サービスへの通信を一切行わない（完全ローカル動作） |

### 3.5 保守性要件

| 要件ID | 項目 | 仕様 |
|--------|------|------|
| NFR-12 | コード構成 | エンジン層・UI層・アプリ層の責務分離 |
| NFR-13 | 外部依存 | フレームワーク等の外部依存なし（Vanilla JS） |
| NFR-14 | テストカバレッジ | Puppeteerによる自動テストで主要機能を網羅 |

## 4. 制約事項

| 要件ID | 制約事項 |
|--------|---------|
| CON-01 | ブラウザの自動再生ポリシーにより、AudioContext はユーザー操作後に初期化が必要 |
| CON-02 | iOS Safari では AudioContext の `resume()` が必要な場合がある |
| CON-03 | バックグラウンドタブでは `setInterval` の精度が低下する可能性があるが、Web Audio API のスケジューリングで緩和 |
| CON-04 | 奇数拍子（3拍子・5拍子）で2分音符モード使用時、小節境界を跨ぐ演奏になる |

## 5. ユースケース

### 5.1 UC-01: 基本的なテンポ練習

**アクター:** 演奏者

**事前条件:** アプリがブラウザで開かれている

**手順:**
1. 希望のテンポをスライダーまたは±ボタンで設定する
2. 拍子を選択する（デフォルト4拍子）
3. STARTボタンを押す
4. 1拍目のアクセント音とインジケーターの強調を確認しながら演奏する
5. 練習終了後、STOPボタンで停止する

### 5.2 UC-02: サブディビジョン練習

**アクター:** 演奏者

**事前条件:** 基本テンポで演奏できる

**手順:**
1. テンポと拍子を設定する
2. 音符モードから「8分音符」「16分音符」等を選択する
3. STARTボタンを押す
4. メイン拍とサブディビジョン音の違いを聞きながら細かいリズムを練習する

### 5.3 UC-03: 変拍子練習

**アクター:** 演奏者

**事前条件:** 拍の感覚が把握できる

**手順:**
1. 拍子を「5拍子」に設定する
2. テンポを設定する
3. STARTボタンを押す
4. 1拍目の強調を基準に、5拍周期のリズムを体得する
